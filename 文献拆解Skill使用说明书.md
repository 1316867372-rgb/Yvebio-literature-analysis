# 学术文献PDF拆解 Skill 使用说明书

---

## 这是什么

一个自动化工具套件。输入一篇英文学术论文 PDF，自动产出：
- 8模块中文拆解文档（docx，含嵌入式图表）
- PDF内嵌Fig图独立提取（PNG）
- 研究流程图谱（PNG，4:3横版信息图）

适用于生物医学领域（肿瘤、信号通路、细胞表型类研究）。

---

## 安装

### 1. 安装 Skill

把 `biomed-literature-analysis.skill` 文件放到：
```
~/.config/opencode/skills/
```

### 2. 安装 Python 依赖

```bash
pip install pdfplumber pymupdf python-docx pillow
```

### 3. （可选）配置信息图自动生成

第5步"研究流程图"需要外部生图工具。如不配置则自动跳过，不影响其余产出。

在系统环境变量中设置：
```
INFOGRAPHIC_TOOL_DIR = D:\你的路径\infographic-tool
```

指向的目录需包含一个 `generate_infographic.py` 模块，对外提供 `process_infographic_task` 函数。该模块由你在营内搭建的 Dify 工作流工具提供，不随本 Skill 分发。

## 使用步骤

### 前置准备

把 PDF 文件和参考规范文件放在同一个文件夹内，目录结构如下：

```
文献解读/
├── 你的论文.pdf                    ← 要拆解的文献
├── 参考要求/                        ← 规范文件夹（必须有）
│   ├── 文献拆解模板.docx
│   ├── 论证规范1-因果逻辑.docx
│   ├── 论证规范2-思考1-理论规范.docx
│   ├── 论证规范3-思考2-文献中的论证.docx
│   ├── 要素与科学假设.docx
│   ├── 研究模型.docx
│   ├── 一句话概述.docx
│   ├── 套路归纳.docx
│   ├── 答案-因果逻辑.docx
│   ├── 答案-图表解读.docx
│   ├── 答案-套路与机制模式.docx
│   ├── 思考-因果逻辑.docx
│   ├── 思考-图表解读.docx
│   └── 思考-套路与机制模式.docx
```

### 触发拆解

在 opencode 对话中输入：

```
拆解文献 "\\192.168.100.200\知识中心\...\你的论文.pdf"
```

工具会自动按以下顺序执行：

| 步骤 | 操作 | 产出 |
|------|------|------|
| 1 | 读取全部14篇参考规范 | 分析依据 |
| 2 | 提取PDF全文 | 文本数据 |
| 3 | 撰写8模块拆解文档 | `论文名_文献拆解_主变量.docx` |
| 4 | 生成AI绘图提示词 | `论文名_研究流程_AI绘图提示词.txt` |
| 5 | 调用生图工具生成流程信息图 | `论文名_研究流程.png`（需配置生图工具） |
| 6 | 从PDF拆出Fig图 | `论文名 Fig.1.png` ~ `Fig.N.png` |
| 7 | Fig图+流程图嵌入docx | 最终完整文档 |

## 8模块文档内容说明

| 模块 | 内容 |
|------|------|
| 一、关键要素 | 疾病/表型/变量分类 + 机制模式 + 研究模型 |
| 二、科学假设 | 嵌套式完整假设句 |
| 三、逻辑关系 | 因果逻辑链逐条列出，附对应图号 |
| 四、知识背景 | 5个维度深度展开，每项3-5段 |
| 五、研究流程 | 按Fig顺序详述实验推进逻辑 |
| 六、数据解读 | 每Fig逐子图详读（目的+分组+方法+结果+结论） |
| 七、论证规范 | 理论规范 vs 实际完成度逐项对照 |
| 八、套路归纳 | 优缺点各≥5条，附具体分析 |

## 常见问题

**Q: Fig图提取数量不对？**
A: 部分PDF的图注和图片不在同一页，自动检测可能漏掉。此时需手动补充提取。属于已知限制。

**Q: 信息图生成了但很简陋？**
A: 极简风格是有意设计，便于学术海报展示。如需更丰富的信息量，修改提示词 txt 中的描述即可。

**Q: 文档字数不够？**
A: 正常应在5000-8000字（纯文本）。如果明显偏少，检查是否所有14篇参考规范都成功读取。

**Q: 可以用于非肿瘤/非生物医学领域吗？**
A: 8模块框架和论证规范是通用生物医学逻辑。但如果文献不涉及分子调控机制，论证规范部分可能不完全适用。
